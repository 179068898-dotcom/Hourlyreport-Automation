# 同事使用说明

版本：v0.4.21（内部修复版）

百度竞价日报 / 小时报自动化工具，内部包已预置五个项目配置和六组百度凭据。其中沈阳牛需要分别读取沈阳中亚、沈阳银康两个百度来源，程序会自动聚合后写入 Excel。

## 开箱步骤

1. 解压到固定目录，如 `D:\自动化脚本\hourly_report_bot`
2. 双击 `run_menu.bat`；第一次会自动安装运行环境，安装完成后自动打开菜单
3. 选择 5. 文件合格校验，确认关键项通过
4. 把商务通导出 Excel/CSV 放到各项目的商务通目录

首次安装需要网络。当前内置项目使用 `openpyxl` 写 Excel，不需要安装 `xlwings`；`xlwings` 仅为以后切换到 Microsoft Excel COM 写入方式时的可选依赖。

## 只需改两个路径

每个项目配置文件在 `configs/projects/项目ID.json`，一般只需改两项：

- `excel.path` — 目标 Excel 文件路径
- `kst.export_dir` — 商务通导出文件目录

其他（项目名称、账户映射、百度账号密码、日报小时报字段）已预置。

## 日常使用

双击 `run_menu.bat`，程序会先检查运行环境；依赖缺失或上次安装中断时会自动重新安装。菜单打开后，顶部显示当前项目，按编号选择：

1. 小时报 → 选时段（11点/15点/18点）→ 确认 → 执行
2. 日报 → 确认 → 执行
3. 项目列表 → 选择要操作的项目
4. 项目信息 → 查看当前项目配置
5. 文件合格校验 → 检查环境是否就绪
0. 退出

运行成功后 Excel 自动打开，菜单显示完成时间。

## 沈阳牛双来源说明

- 沈阳牛小时报、日报都会依次读取两个百度来源，再合并为统一结果。
- 日报最终仍使用 `reports/baidu_daily_data.json` 进入后续合并和写入流程。
- 任一百度来源读取失败时，本次任务会中断，不会继续写 Excel。
- 诊断时查看 `reports/baidu_multi_source_report.json`、`reports/baidu_multi_source_report.md` 和 `logs/run.log`。
- 报告中的候选账户如果未映射到 Excel 写入区域，不会被自动写入。

## 注意事项

- 写入前关闭 WPS/Excel 中的目标表格
- 内部包包含 `secrets/secrets.json`，不要发送给外部或上传到公共位置
- 商务通导出文件的对话计数字段支持 `访客消息数`、`访客发送消息数` 或 `访客发送数`，三者统计口径相同
- 不要手动编百度数据
- JSON 路径用 `/`，不要用单个 `\`

## 双击没反应

1. 确认已解压全部文件
2. 在文件夹地址栏输入 `cmd` 回车，运行 `run_menu.bat`
3. 如果安装时仍提示 `xlwings` 无可用版本，说明使用的是旧 `v0.4.19` 发布包；请换用 `v0.4.21` 修复包后重新解压运行
4. 把黑色窗口的报错截图发回来

# 数据模式菜单与安全停止控件设计

## 目标

在不改变现有业务命令、Excel 写入边界和窗口尺寸的前提下，完成三项 GUI 调整：

1. 标题栏新增“数据模式”菜单，将 A/B 数据源控件从“系统配置”菜单迁移到该菜单。
2. 小时报、日报主操作改为 7:3 的运行/停止斜切组合按钮。
3. 全局字体按标题高频程度组合使用微软雅黑加粗与微软雅黑 Light。

## 数据模式菜单

- “数据模式”与“系统配置”在标题栏并排，字号、悬浮状态和高度一致。
- 点击“数据模式”打开独立白色浮层，再次点击关闭；不使用悬浮自动展开。
- 浮层只放 A/B 数据源控件，不混入其他系统配置项。
- A/B 控件缩为 `106 x 29`，与“单项目 / 多项目”分段控件同尺寸。
- B 表示强制浏览器，A 表示 API 优先；继续复用现有配置持久化、失败回滚和任务期间锁定逻辑。
- 打开一个标题栏菜单时，关闭另一个标题栏菜单，避免浮层重叠。

## 运行/停止组合按钮

- 小时报和日报各使用一个固定尺寸组合控件；整体占用原主按钮位置，不改变卡片和窗口尺寸。
- 左侧约 70% 为“运行小时报”或“运行日报”，右侧约 30% 为“停止”。
- 中间边界使用向右上倾斜的白色空白线，不使用垂直分割线。
- 停止区相对运行区向左、向下偏移 1-2 像素，形成轻微错层，但点击区域不得重叠或越界。
- 空闲时停止区禁用并显示灰色；任务启动后，仅当前小时报或日报对应的停止区启用。
- 项目配置检查、环境安装等非小时报/日报任务运行时，两个停止区均保持禁用。
- 点击停止不弹确认，立即请求终止当前任务，并立刻禁用停止区，防止重复点击。
- 接收到 `excel` 阶段标记时立即禁用停止区；从此直到任务结束都不可停止。
- 用户停止后的退出码不按普通失败展示：状态显示“已停止”，不打开 Excel，不提示任务失败。
- 正常完成、启动失败或普通失败均恢复运行按钮，并保持停止按钮禁用。

## 字体层级

- 基础正文使用 `Microsoft YaHei Light`，后备顺序为 `Microsoft YaHei UI`、`Microsoft YaHei`、`Segoe UI`、`sans-serif`。
- 产品标题、卡片标题、当前流程标题、实时日志标题等高频大标题使用 `Microsoft YaHei UI` 的 DemiBold / CSS `600`。
- 项目、日期、时段、运行/停止等主要操作使用常规微软雅黑，不使用过粗字重。
- 系统配置、数据模式、说明文字、状态时间、菜单次级项等低频内容使用 Light。
- 日志区域继续使用 Consolas / Cascadia Mono，不改为微软雅黑。

## 安全与测试

- 不执行真实 `run` / `run-daily`，不写 Excel。
- 测试覆盖数据模式菜单位置与尺寸、A/B 持久化和锁定、组合按钮 7:3 几何、停止状态机、Excel 阶段锁定、用户停止结果以及字体层级。
- 相关测试通过后运行全量 `tests/test_basic.py`，再重建单文件 EXE 并做真实窗口烟雾检查。

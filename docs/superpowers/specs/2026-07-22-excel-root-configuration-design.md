# Excel 根目录配置设计

## 目标

在系统菜单提供“Excel 路径配置”，用户只需选择本机的 `【竞价】` 文件夹，程序即可批量迁移九个正式项目的 Excel 路径。

## 规则

- 仅处理 `configs/projects` 中非模板、非演示的正式项目。
- 每个旧路径必须包含名称完全一致的 `【竞价】` 路径段。
- 新路径由“用户选择的 `【竞价】` 文件夹 + 旧路径中 `【竞价】` 后的相对路径”组成。
- 所有目标 Excel 必须存在且扩展名为 `.xlsx`、`.xlsm` 或 `.xls`。
- 任一项目校验失败时，不备份、不写入任何项目配置。
- 全部校验通过后，先把全部原配置复制到带时间戳的备份目录，再逐个原子写入。
- 只修改 `excel.path`，不修改项目其他字段、模板、商务通路径和账号配置。
- 成功后刷新 GUI 项目数据并显示修改数量与备份位置；失败时列出项目名和预期路径。

## UI

系统浮层菜单新增独立行“Excel 路径配置”。点击后打开文件夹选择器，选择器用于选择名称为 `【竞价】` 的目录。取消选择不产生任何修改。

## 安全性

批量写入采用全部校验后再执行。写入过程中若发生异常，使用本次备份恢复已改动文件，避免出现部分项目使用新路径、部分项目使用旧路径的状态。

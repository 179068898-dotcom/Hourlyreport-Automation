# 多项目执行设计

## 目标

在不改变单项目生产路径的前提下，为 GUI 增加 1 至 3 个项目的多项目执行能力。百度 API 数据并行读取，快商通解析、数据合并和 Excel 写入按用户选择顺序串行执行。

## 不变约束

- 单项目仍调用现有 `run` / `run-daily`，保留 API 优先和浏览器降级。
- 多项目只允许 API，不启动 Chrome，不降级浏览器，也不混用两种来源。
- 每个 Excel 写入前继续使用现有备份、结构识别、保护元数据恢复和写后复核。
- 多项目最多 3 个、最少 1 个；重复项目、未知项目、重复 Excel 路径在开始前拒绝。
- 本阶段不改版本号、不构建发布包。

## 执行模型

### 阶段一：并行 API 准备

协调器一次提交全部所选项目到固定大小线程池。每个项目使用现有 20 秒 API 自修复预算；单来源和双来源项目都必须完整成功。输出写到本次运行的独立目录：

```text
reports/multi_runs/<run_id>/<project_id>/baidu_account_data.json
reports/multi_runs/<run_id>/<project_id>/baidu_daily_data.json
```

API 失败只标记当前项目失败，不影响其他项目。多项目 API 函数没有浏览器回调入口。

### 阶段二：串行落盘

成功取得 API 数据的项目按用户选择顺序依次进入现有流水线。流水线的百度抓取步骤由缓存结果替代，后续仍调用现有快商通解析、合并和 Excel 写入函数。任一项目失败后记录原因并继续下一个项目。

共享的 `reports/*.json` 只会在串行阶段由一个项目使用，因此不存在并发覆盖。每个项目完成后，把完整返回报告写入本次运行的项目目录，最终汇总写入：

```text
reports/multi_project_run_report.json
```

### 停止语义

GUI 为多项目任务创建独立的队列停止文件。点击停止只提出“不要开始下一个项目”的请求：

- 当前正在执行的项目继续完成，包括当前项目的 Excel 写入。
- 已完成 API 但尚未进入串行流水线的项目标记为 `stopped`，不写 Excel。
- 已经启动的 API 请求允许自然结束，不强制终止线程。
- 单项目继续使用原有 Excel 前停止门，不受影响。

### 完成与打开 Excel

多项目协调器本身正常结束时返回 0，即使部分项目业务失败；致命参数、配置或调度异常才返回非 0。GUI 读取汇总报告显示“全部完成”“部分完成”或“已停止”。全局 Excel 自动打开启用时，在全部串行写入结束后一次性打开所有成功项目 Excel，去重后按选择顺序打开。

## 选择持久化

本地文件 `configs/multi_project_selection.json` 保存有序项目 ID 列表，原子写入。切换到多项目模式时恢复最后一次有效选择；已删除项目自动过滤；全部失效时回退当前默认项目。该文件属于本机配置，不进入在线更新覆盖范围。

## 日志与状态

控制台输出统一带项目和阶段前缀：

```text
[多项目][API 1/3][昆明牛] 开始读取百度数据
[多项目][写入 1/3][昆明牛] 开始串行处理
[多项目][失败][宁波牛] authorization_error
[多项目][完成] 成功 2，失败 1，停止 0
```

最终报告至少包含 `run_id`、任务类型、日期/时段、选择顺序、每项目状态、错误、Excel 路径、开始/结束时间和汇总计数。

## 验证重点

- API 调用确实重叠，Excel 写入绝不重叠且保持选择顺序。
- 一个项目 API 失败不影响其他项目写入。
- 双百度来源任一路失败时该项目不进入 Excel 阶段。
- 停止不会中断当前项目，只阻止后续项目。
- 单项目命令、停止和浏览器降级回归不变。
- 测试只使用临时 JSON 和依赖注入，不触碰真实 Excel、Chrome 或百度 API。

# 快商通后台规则

## 目标字段

| Excel 字段 | 统计口径 |
|---|---|
| 总对话 | 当前账户、当前日期、当前时段下 `访客消息数 >= 1`、`访客发送消息数 >= 1` 或 `访客发送数 >= 1` 的对话数 |
| 有效 | 名片标签包含 `转潜-有效`、`有效-一般`、`有效-三句` |
| 有效转潜 | 名片标签包含 `转潜-有效` |
| 总转潜 | 名片标签包含 `转潜-` |

## 固定规则

```text
访客消息数 / 访客发送消息数 / 访客发送数 = 商务通导出的等价表头，字段值 >= 1 才视为一条对话
有效对话 = 名片标签包含“转潜-有效”或“有效-一般”或“有效-三句”的对话
有效转潜 = 名片标签包含“转潜-有效”的对话
总转潜 = 名片标签包含“转潜-”的对话
```

## 对话账户归属

快商通数据必须能归属到标准账户：

- 银康01
- 银康银屑02
- 银康03

无法归属的对话进入异常名单，不写入正式账户数据。

## 自检

- 有效 ≤ 总对话；
- 有效转潜 ≤ 有效；
- 总转潜 ≤ 总对话；
- 同一对话不能重复统计；
- 名片标签为空的对话只进入总对话，不进入有效和转潜。

```json
{

  "period": "15点",
  "accounts": {

    "银康03": {"总对话": 0, "有效": 0, "有效转潜": 0, "总转潜": 0}
  },

```

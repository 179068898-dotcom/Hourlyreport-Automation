# GUI 多项目选择预留方案

> 状态：2026-07-13 选定 A 版并完成 GUI 选择层；批量执行队列尚未接入，当前单项目运行逻辑不变。

## 设计目标

- 默认仍为单项目模式，现有用户无需重新学习。
- 未来批量模式支持选择 2 个或更多项目，按队列顺序依次执行。
- 选择阶段禁止重复项目；只选 1 个项目时不允许启动批量任务。
- 不在项目框内堆满标签，避免固定宽度界面被撑高或截断。

## 推荐交互

1. 在“项目”标题右侧使用紧凑分段控件：`单项目 | 多项目`。
2. 单项目模式完全沿用当前下拉框。
3. 批量模式点开后显示与项目框等宽的白色浮层：
   - 顶部：搜索项目（项目超过 8 个时显示）。
   - 中部：项目列表，每行包含复选框、项目名、项目 ID 和配置状态点。
   - 底部：`已选 N 个`、`清空`、`完成`。
4. 勾选项目后浮层保持打开，直到点“完成”、按 `Esc` 或点击浮层外部。
5. 收起状态的显示规则：
   - 1 个：显示完整项目名（仅单项目模式）。
   - 2 个：显示第一个项目名和 `+1`。
   - 3 个及以上：显示 `已选择 N 个项目`。
6. 多项目模式少于 2 个项目时不允许确认选择，并显示“至少选择 2 个项目”。
7. 执行队列接入前，多项目模式点击运行或配置检查只显示说明，不得降级为仅执行第一个项目。

## 执行策略预留

- 第一版只做顺序队列，不做浏览器并行；继续复用当前稳定的 Chrome、登录、抓数和 Excel 写入链路。
- 项目顺序默认采用配置列表顺序，后续可增加拖动排序，但不作为首版需求。
- 每个项目保持独立日志、报告和 Excel 路径；一个项目失败后记录结果，再继续下一个项目。
- 任务卡显示总进度：`2 / 4 · 正在执行长沙牛`。
- 完成后展示批量结果摘要：成功数量、失败数量、每个项目的报告入口。

## 视觉规格

- 浮层宽度与项目框一致，圆角 10px，无系统阴影，边框使用当前 `#cad7e8`。
- 列表行高 34px；勾选状态使用绿色线框和代码绘制对勾。
- 项目名使用正文 9pt，项目 ID 使用次级 8pt 灰色文字。
- 选中摘要使用一个紧凑数量标签，不连续堆叠多个项目标签。
- 键盘支持：上下方向键移动、空格切换、`Enter` 完成、`Esc` 关闭。

## 参考依据

- Microsoft Fluent 2 多选 Combobox：每个选项带复选框，多选时列表保持打开，并建议在输入框中用标签展示已选项。
  - https://fluent2.microsoft.design/components/web/react/core/combobox/usage
- Ant Design Select：支持 multiple 模式、限制可见标签数量和搜索，适合固定宽度下的多选摘要。
  - https://4x.ant.design/components/select/
- WAI-ARIA Listbox：多选列表需暴露多选状态，并为每个选项提供明确的选中状态。
  - https://www.w3.org/WAI/ARIA/apg/patterns/listbox/

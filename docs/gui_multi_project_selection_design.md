# GUI 多项目执行说明

## 选择规则

- 默认仍为单项目模式。
- 多项目模式允许选择 1 至 3 个项目，按选择先后顺序显示和执行。
- 选择会保存到 `configs/multi_project_selection.json`，下次切回多项目时自动恢复。
- 已删除或停用的项目会自动过滤；没有有效历史选择时回退当前默认项目。
- 重复项目、未知项目、超过 3 个项目或多个项目指向同一个 Excel 时拒绝启动。

## 执行规则

1. 所有选中项目并行读取百度 API，分别使用现有 Token 自修复和完整性校验。
2. 多项目模式不启动 Chrome，不做浏览器降级。某项目 API 失败时，只跳过该项目。
3. API 成功项目按选择顺序逐个解析快商通、合并数据、备份并写入 Excel。
4. 沈阳牛、沈阳白的双百度来源必须两路同时成功；任一路失败则整个项目失败。
5. 所有项目结束后输出 `reports/multi_project_run_report.json`，包含成功、失败和停止明细。
6. 开启 Excel 自动模式时，全部写入结束后按选择顺序一次性打开成功项目的 Excel。

## 停止规则

- 点击停止不会中断当前项目，也不会破坏正在进行的 Excel 写入。
- 当前项目完成后，不再开始下一个排队项目。
- 已完成 API、但尚未写入的项目标记为“停止”，临时数据不会进入 Excel。
- 单项目模式继续使用原来的“Excel 写入前安全停止”规则。

## 日志格式

```text
[多项目][API 1/3][昆明牛] 开始读取百度数据
[多项目][写入 1/3][昆明牛] 开始串行处理
[多项目][失败][宁波牛] authorization_error
[多项目][完成] 成功 2，失败 1，停止 0
```

# v2026.7.22.108 更新说明

## 本版重点

- 系统菜单新增 `Excel 自动打开`，可统一控制小时报和日报完成后是否打开 Excel。
- 系统菜单新增 `Excel 路径配置`，选择本机的 `【竞价】` 文件夹后，自动匹配全部正式项目的固定后续路径。
- Excel 路径配置采用全量校验、写前备份和失败回滚；任何一个项目无法匹配时，所有项目均不修改。
- 新增顶部 `帮助` 菜单。
- `帮助 -> 关于小螃蟹` 增加 Codex 风格版本信息弹窗。
- `帮助 -> 检查版本` 支持手动触发 GitHub Release 更新检测。
- 在线更新请求改用程序内置 CA 证书，修复部分新电脑的 HTTPS 证书校验失败。
- 启动时检查更新失败不再显示误导性的 `重试`；本地版本不低于线上版本时自动隐藏更新按钮。

## 使用说明

- `Excel 自动打开` 默认启动，同时作用于小时报和日报。
- 配置 Excel 路径时必须选择名称正好为 `【竞价】` 的文件夹，不要选择其上级或下级目录。
- 系统会先确认九个项目的目标 Excel 全部存在，再备份并更新本地项目配置。
- 手动检查版本不会自动下载；检测到新版本后仍需点击顶部更新按钮。

## 发布资产

- 完整安装器：`Hourlyreport_automation_setup_v2026.7.22.108.exe`
- 在线更新包：`Hourlyreport_automation_v2026.7.22.108.zip`

## 验证重点

- GUI 可以正常启动。
- 关闭 `Excel 自动打开` 后，小时报和日报完成均不自动打开 Excel。
- Excel 路径配置可以一次匹配九个项目，缺少任一文件时保持原配置不变。
- 帮助菜单可以打开关于窗口。
- 手动检查版本可以触发现有在线更新检查流程。
- HTTPS 证书异常的电脑可以正常访问 GitHub Release 更新接口。

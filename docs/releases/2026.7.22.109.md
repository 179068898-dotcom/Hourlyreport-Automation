# v2026.7.22.109 更新说明

## 本版重点

- 多项目模式正式支持同时选择 1 至 3 个项目，并记住最后一次选择顺序。
- 多项目百度数据通过 API 并行读取，快商通解析、数据合并和 Excel 写入按选择顺序串行执行。
- 多项目模式不降级浏览器；单个项目 API 或配置失败时跳过该项目，其他项目继续完成。
- 点击停止不会中断当前项目，只阻止下一个排队项目开始。
- 所有项目结束后统一汇总成功、失败和停止结果，并按 Excel 自动打开设置打开成功项目文件。
- 修复在线更新点击“重启”后旧客户端退出，但更新助手因缺少 PySide6 而静默失败的问题。
- 打包版改为复制主 EXE 自身作为临时更新助手，不再依赖项目 Python 单独安装 GUI 组件。
- 更新窗口完成初始化后才允许旧客户端退出；新版启动后会检查进程存活，失败时最多重试 3 次并记录日志。
- 重塑 GitHub 项目首页，新增品牌横幅、真实 GUI 截图、能力矩阵、单项目/多项目流程图和折叠式使用说明。

## 多项目规则

- 最少选择 1 个、最多选择 3 个项目。
- 重复项目、重复 Excel 路径在 API 请求前拒绝启动。
- 沈阳牛、沈阳白的双百度来源仍要求两路全部成功后才进入 Excel 阶段。
- 汇总报告：`reports/multi_project_run_report.json`。
- 每次运行的独立 API 数据：`reports/multi_runs/<run_id>/<project_id>/`。

# 阶段开发计划

## 第一阶段：Excel 表结构识别

命令：

```cmd
python main.py --mode inspect-excel
```

目标：

- 不打开百度后台；
- 不打开快商通；
- 不写入 Excel；
- 只扫描 `时段数据` sheet；
- 识别账户区域和字段表头；
- 输出 `reports/excel_structure_report.json` 与 `reports/sheet_text_dump.csv`。

验收标准：

- 找到银康01、银康银屑02、银康03 三个账户区域；
- 找到每个账户区域内部的日期、时段、展现、点击、消费；
- 尽量找到总对话、有效、有效转潜、总转潜；
- 找不到时输出疑似标题，不猜坐标。

## 第二阶段：模拟写入 Excel

命令：

```cmd
python main.py --mode mock-write --period 15点
```

目标：

- 使用模拟数据写入测试 Excel；
- 写入前备份；
- 只写三个账户分户区域；
- 不改汇总区域；
- 写入后复核。

## 第三阶段：百度后台数据读取

命令：

```cmd
python main.py --mode fetch-baidu --period 15点
```

目标：

- 使用 Playwright 控制浏览器；
- 如果未登录，停下来等待人工登录；
- 读取三个账户展现、点击、消费；
- 输出 `reports/baidu_account_data.json`；
- 不写 Excel。

## 第四阶段：快商通后台数据读取

命令：

```cmd
python main.py --mode fetch-kst --period 15点
```

目标：

- 使用 Playwright 控制快商通后台；
- 如果未登录，停下来等待人工登录；
- 读取对话数据；
- 按账户统计总对话、有效、有效转潜、总转潜；
- 输出 `reports/kst_dialog_data.json`；
- 不写 Excel。

## 第五阶段：整合写入

命令：

```cmd
python main.py --mode run --period 15点
```

目标：

- 读取百度数据；
- 读取快商通数据；
- 合并账户数据；
- 写入 Excel；
- 保存；
- 输出写入报告和自检报告。

# 给 Codex 的开局指令

复制下面这段给 Codex：

```text
请全程用中文回复。

请先读取当前项目下的 AGENTS.md、README.md 和 docs 目录中的规则文件。

当前项目是 Windows 本地运行的“百度竞价日报/小时报自动化工具”。
当前版本已经删除 QQ 和截图功能，只做：
1. 从百度营销后台读取三个账户的展现、点击、消费；
2. 从快商通后台读取三个账户的总对话、有效、有效转潜、总转潜；
3. 写入本地 Excel 文件的“时段数据”sheet；
4. 保存 Excel；
5. 输出日志和自检报告。

请严格按阶段开发，不允许一上来做全流程。

当前只做第一阶段 Excel 表结构识别：
不打开百度后台，不打开快商通，不写入 Excel。

第一阶段目标：
读取 samples 目录里的测试 Excel，打开 sheet“时段数据”，自动识别：
1. 每日时段统计数据 / 汇总区域；
2. 银康01 数据区域；
3. 银康银屑02 数据区域；
4. 银康03 数据区域；
5. 每个账户区域中的日期列、时段列、展现列、点击列、消费列、总对话列、有效列、有效转潜列、总转潜列；
6. 输出识别结果到 reports/excel_structure_report.json；
7. 输出全部非空单元格扫描结果到 reports/sheet_text_dump.csv；
8. 输出运行日志到 logs/run.log。

先帮我检查当前项目框架是否完整，然后告诉我：
1. 需要我把测试 Excel 放到哪个位置；
2. config.json 应该怎么配置；
3. 第一阶段应该运行什么命令；
4. 如果识别不到账户，应该看哪个报告文件。

确认后再运行第一阶段，不要直接进入写入和后台自动化。
```

# Textual 终端工作台使用说明

## 启动

在项目目录中双击或执行：

```cmd
run_textual.bat
```

该入口使用项目 `.venv` 启动 `textual_app.py`，并预先设置 UTF-8 控制台环境。原有 `run_menu.bat` 继续保留并独立可用。

## 可执行功能

Textual 工作台提供以下手动操作：

- 运行小时报：分别启动 `11点`、`15点`、`18点` 小时报批处理入口。
- 运行日报：启动昨日日报，或输入 `YYYY-MM-DD` 后运行指定日期日报。
- 预检：分别检查小时报条件和日报条件。
- 凭据检查：调用既有凭据状态检查命令。
- `doctor` 与项目配置校验。
- 查看 `reports` / `logs` 目录。
- 在底部日志区查看命令输出、错误输出和返回码。

工作台不会按当前时间自动启动任务，所有任务均由操作人员点击按钮发起。
日报或小时报只有在最终报告显示写入复核通过且确有写入后，才会在状态区标记为完成。

## 快捷键

- `q`：退出。
- `r`：刷新只读状态。
- `c`：清空界面日志区。

## 安全边界

- 不保存密码。
- 不显示密码。
- 不替代 run_menu.bat。
- 不重写业务逻辑，日报和小时报仍调用既有批处理入口。
- 首页状态仅读取现有配置、任务状态和已有检查报告缓存，不自动连接百度、不自动运行检查、不写 Excel。
- 执行失败时查看工作台日志区及 `reports` / `logs` 中的报告与日志。

## 与 CMD 菜单的区别

CMD 菜单仍是稳定入口；Textual 工作台是在同一套业务命令之上的独立终端界面，便于集中查看状态、执行任务和跟踪输出。
